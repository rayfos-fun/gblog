在程式开发的世界裡，「编辑器之战」是永恆的圣战。有人拥护 Vim 的上古神力，有人热爱 VS Code 的现代化与丰富生态。

身为一个（自称）老派的程式员，我个人始终坚持使用 **Sublime Text**。

在我们开始之前，先来个免责声明：**这不是要战IDE，也请不要乱喷。**

### 哲学时刻：没有「最好」，只有「最适合」

我始终坚信，所有事情都是主观的「trade-off」（取捨），从来没有客观的绝对第一名。

一个工具是否「好用」，其中包含的面向千千万万：
* **价钱**：免费、付费，还是一次买断？
* **随处可得**：是否支援我所有的平台？
* **直觉易上手**：我需要花多少时间学习？
* **使用习惯**：它是否符合我的肌肉记忆？
* **高度客製化**：它能让我「为所欲为」吗？

每个人的心中，衡量这些的尺都长得不一样。只要是有人真心拥护的玩意，那它肯定有值得拥护的原因。请尊重他人的选择。

### 为何是Sublime？因为「无限的可能性」

对我而言，选择Sublime Text的关键原因只有一个：**我可以用Python轻鬆自製Plugin。**

我不需要的功能，它可以完全消失；我想要的功能，只要我写得出来，它就能实现。这给了我「无限的可能性」。

与其说我在使用软体，不如说我在「打造」我的软体。

### 秀一下：那些「全世界只有我有」的独门功能

光说不练太空泛。对我而言，一个完美的编辑器，就是要能融入我的工作流，成为我大脑的延伸。以下是我自製的几个最常用功能：

#### 1. 万用「上下文感知」快速连结

我设定了一个快捷键 `Ctrl+O`。它的功能完全取决于我指标下的文字：

* 如果指标指在 "task" 这段文字上，按下 `Ctrl+O`，Sublime会立刻帮我开启 `.\task.txt` 这个档案。
* 如果指标指在 "http" 等超连结文字上，按下 `Ctrl+O`，它会立刻呼叫Chrome浏览器，开启网址。
* 如果指标指在 "import mylog from mylib" 上，按下 `Ctrl+O`，它会立刻开启正确的mylog.py档案。

我不需要切换视窗、不需要去点书籤，我需要的一切，都在`Ctrl+O`。

#### 2. 随心所欲的「快速插入/取代」

另一个快捷键 `Ctrl+I`，是我用来插入或取代文字的魔法棒：

* 在任何地方，只要指标指在小写 `d` 上，按下 `Ctrl+I`，它会立刻被取代成今天的日期，例如：`2025-11-12`。
* 如果指标指在 `t` 上，它会被取代成现在的时间，例如：`17:33:25`。

#### 3. 智能「起手式」范本

当我开启一个空白档案，如果我将它储存成 `.py` 结尾（例如 `test.py`），我的自製脚本会立刻侦测到，并自动插入Python的「起手式」：

```python
#!/usr/bin/env python
# -*- coding: utf-8 -*-

def main():
    pass

if __name__ == '__main__':
    main()
```

这节省了我每天无数次的重複打字时间。

### 结语：当工具成为你的一部分

对我而言，这不仅仅是为了省下几秒钟。

当你拥有一项渴望的功能，而你使用的软体能提供无限的扩充性，并且你「有能力」去客製化它——这个过程，除了让你得到一个「完美契合自己」的软体外，更满足了身为开发者的一丝丝卓越感。

「这个功能，全世界只有我有。」

这，就是老派程式员的浪漫。
