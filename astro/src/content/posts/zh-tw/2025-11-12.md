在程式開發的世界裡，「編輯器之戰」是永恆的聖戰。有人擁護 Vim 的上古神力，有人熱愛 VS Code 的現代化與豐富生態。

身為一個（自稱）老派的程式員，我個人始終堅持使用 **Sublime Text**。

在我們開始之前，先來個免責聲明：**這不是要戰IDE，也請不要亂噴。**

### 哲學時刻：沒有「最好」，只有「最適合」

我始終堅信，所有事情都是主觀的「trade-off」（取捨），從來沒有客觀的絕對第一名。

一個工具是否「好用」，其中包含的面向千千萬萬：
* **價錢**：免費、付費，還是一次買斷？
* **隨處可得**：是否支援我所有的平台？
* **直覺易上手**：我需要花多少時間學習？
* **使用習慣**：它是否符合我的肌肉記憶？
* **高度客製化**：它能讓我「為所欲為」嗎？

每個人的心中，衡量這些的尺都長得不一樣。只要是有人真心擁護的玩意，那它肯定有值得擁護的原因。請尊重他人的選擇。

### 為何是Sublime？因為「無限的可能性」

對我而言，選擇Sublime Text的關鍵原因只有一個：**我可以用Python輕鬆自製Plugin。**

我不需要的功能，它可以完全消失；我想要的功能，只要我寫得出來，它就能實現。這給了我「無限的可能性」。

與其說我在使用軟體，不如說我在「打造」我的軟體。

### 秀一下：那些「全世界只有我有」的獨門功能

光說不練太空泛。對我而言，一個完美的編輯器，就是要能融入我的工作流，成為我大腦的延伸。以下是我自製的幾個最常用功能：

#### 1. 萬用「上下文感知」快速連結

我設定了一個快捷鍵 `Ctrl+O`。它的功能完全取決於我指標下的文字：

* 如果指標指在 "task" 這段文字上，按下 `Ctrl+O`，Sublime會立刻幫我開啟 `.\task.txt` 這個檔案。
* 如果指標指在 "http" 等超連結文字上，按下 `Ctrl+O`，它會立刻呼叫Chrome瀏覽器，開啟網址。
* 如果指標指在 "import mylog from mylib" 上，按下 `Ctrl+O`，它會立刻開啟正確的mylog.py檔案。

我不需要切換視窗、不需要去點書籤，我需要的一切，都在`Ctrl+O`。

#### 2. 隨心所欲的「快速插入/取代」

另一個快捷鍵 `Ctrl+I`，是我用來插入或取代文字的魔法棒：

* 在任何地方，只要指標指在小寫 `d` 上，按下 `Ctrl+I`，它會立刻被取代成今天的日期，例如：`2025-11-12`。
* 如果指標指在 `t` 上，它會被取代成現在的時間，例如：`17:33:25`。

#### 3. 智能「起手式」範本

當我開啟一個空白檔案，如果我將它儲存成 `.py` 結尾（例如 `test.py`），我的自製腳本會立刻偵測到，並自動插入Python的「起手式」：

```python
#!/usr/bin/env python
# -*- coding: utf-8 -*-

def main():
    pass

if __name__ == '__main__':
    main()
```

這節省了我每天無數次的重複打字時間。

### 結語：當工具成為你的一部分

對我而言，這不僅僅是為了省下幾秒鐘。

當你擁有一項渴望的功能，而你使用的軟體能提供無限的擴充性，並且你「有能力」去客製化它——這個過程，除了讓你得到一個「完美契合自己」的軟體外，更滿足了身為開發者的一絲絲卓越感。

「這個功能，全世界只有我有。」

這，就是老派程式員的浪漫。
